## Пререквизитс
* оллама (https://ollama.com/download)
* модель llama3.2:1b (```ollama pull llama3.2:1b```)
* питоновские зависимости в виртуальной среде (```pip install -r requirements.txt```)
  * до этого, соответственно, создать виртуальную среду (```python -m venv .venv```)
* не забыть запустить олламу! (```ollama serve```)

## Что делает
Этот маленький скрипт предназначен для классификации тона отзывов на фильмы. В папке review вы можете видеть примеры отзывов, они (кроме 7.txt, примера некорректных данных) были взяты из [датасета](https://www.kaggle.com/datasets/pawankumargunjan/imdb-review/data). Модель может определить тон как _negative_, _positive_ или _invalidData_, тона _neutral_ нет, потому что взятый датасет его не подразумевает. Модель запускается локально на железе, но была самая легковесная модель в 1b параметров. К сожалению, эта модель не работает с русским языком.

## Как запустить
Установив все вещи из пререквизитс, зайти в виртуальную среду, прописать ```python main.py``` в корне проекта. Файлы с расширением .txt из папки reviews будут классифицированны в файле answer.csv. Если тестовые файлы не были изменены, answer.csv будет совпадать с предложенным answer_example.csv
